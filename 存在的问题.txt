三、总结：
1、CSS中，text不能作为text的“直接子代”。写成 .text-parent > .text-child 不行，.text-parent .text-child 就可以。
2、text要设置成 inline-block 等 ，才能进行 宽、高的 设置
3、image是行内块元素，可以在 其父元素 写上 text-align: center ，使得该图片 水平居中。
4、input输入框用的是 paddng-left ，而不是 text-indent ，这样输入是就是会存在 显示 问题（比如输入关键字与“搜索icon”重叠，参照 search组件）。
5、组件放在 哪个页面， 那当前的相对路径就是 “以该页面为基准的！！？？”
6、require 不支持绝对路径！！！
7、button y语义化好是好，但是自带默认样式太多，写样式 不如 view 爽呀。。。

二、总结
1、图片和文字同一行显示：
让它们都在同一个父节点下面，然后都在 它们的 css s样式里面写上  vertical-align: middle,然后图片要加上 宽、高、padding 等，文字不用、只需要 vertical-align: middle 。
2、父 text-align: center。其子 为image、text等都好会居中显示！！不只是“文字标签”能居中！
3、改变 data 对象的某一属性值单引号 ==》 'sale_list.index'
4、孩子选择器，“后代、即孩子不纯种时”，last-child 会失效，但是 first-child永远不会失效！！
5、商品详情页的 底部购买按钮 颜色应该是渐变的。
6、好像 wx.request真的存在异步问题，会报错 “Setting data field "my_point" to undefined is invalid.”，好像有时网络快了，就不会搞错了。==》 “只要网速够快，bug就跟不上我！！”
7、像 img等非 block元素，想要水平居中 ，则css写成 ==》 display：block; margin: 0 auto;
8、抽象出 请求函数，但其返回结果传不到 page!! ==》
9、箭头函数中 无法访问到 this值问题 ==》 全局中定义一个 that ，并在onload中赋值(that=this),
   注：
    箭头函数体内的this对象，就是定义时所在的对象，而不是使用时所在的对象
    箭头函数不可以当作构造函数，也就是说，不可以使用new命令，否则会抛出一个错误
    箭头函数不可以使用arguments对象，该对象在函数体内不存在。如果要用，可以用Rest参数代替
    不可以使用yield命令，因此箭头函数不能用作Generator函数。
10、选项卡，即使选的跟之前 一样也会触发 change函数，为了 提高性能与优化，放入了 pre_index 这个域。但是该发的图片请求还是发了，是个问题！！
  注：不要在if else while for循环等语句中去定义一个 函数声明 ，定义 函数表达式 和 调用函数 都没有问题。
11、回调函数，应该是 纯小写 function 。严格等 === 。 typeof callbackSuccess === 'function' 
12、“等号传参数，必须放在 在后面一个、而且前面参数必须依次填充完”！！ ==》  util.myRequest('searchByKeyword.php', { keyword: keyword }, 'GET', callbackSuccess = callbackSuccess)
